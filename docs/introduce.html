Vibe Diagnosis

바이브코딩 프로젝트를 위한 자가 진단 프레임워크.
기존 디버깅의 대안으로, 코드가 스스로
"나 지금 정상이야"를 증명합니다.

···

왜 만들었나

기존 디버깅은 문제가 발생한 후에 대응합니다. console.log를 찍고, 스택 트레이스를 추적하고, 고쳤더니 다른 곳이 깨지는 반복.

AI 에이전트가 코드를 대신 짜주는 시대에도 이 문제는 여전합니다. AI가 빠르게 고쳐주지만, "이 프로젝트가 지금 전체적으로 정상인가?"를 아는 방법이 없습니다.

기존 디버깅
에러 발생 →
원인 추적 →
수정 →
다른 곳이 깨짐 →
다시 반복 🔄
자가 진단
작업 완료 →
자동 검증 →
문제 즉시 확인 →
AI 자동 수리 →
전체 재검증 ✅

Vibe Diagnosis는 이 문제에 대한 답입니다. 작업을 완료할 때마다 검증 코드를 함께 만들어서, 프로젝트가 항상 자신의 상태를 증명할 수 있게 합니다.

···

핵심 원칙

🔗 Task ↔ Diagnostic 1:1 매핑

작업(Task)이 완료되면, 그 작업이 정상 동작하는지 검증하는 진단 파일(.diag.js)이 함께 생성됩니다. 코드를 짰으면, 증명도 함께.

3단계 진단 레이어
TASK작업의 의도가 달성되었는가?
FUNC핵심 함수가 올바른 출력을 생성하는가?
SYS외부 서비스 연결과 인프라가 정상인가?

시작하기

AI 도구의 MCP 설정에 아래 JSON을 추가하면 됩니다.

{ "mcpServers": { "vibe-diagnosis": { "command": "npx", "args": ["-y", "vibe-diagnosis-mcp"] } } }

Gemini(Antigravity), Claude Desktop, Cursor, Windsurf에서 사용할 수 있습니다.

설정 후 AI에게 "자가진단 적용해줘" 라고 말하면 초기화, 진단 파일 생성, 대시보드 실행까지 자동으로 진행됩니다.

CLI로 사용하기
npx vibe-diag init # 초기화 npx vibe-diag run # 진단 실행 npx vibe-diag dashboard # 웹 대시보드 npx vibe-diag config get # BYOK 설정 확인 npx vibe-diag repair --all # AI 자동 수리
···

사용 채널

4가지 채널에서 동일한 진단을 실행할 수 있습니다.

🤖MCPAI 에이전트 내에서 실행무료
🖥️대시보드웹 브라우저에서 시각적 관리BYOK
⌨️CLI터미널에서 실행, CI/CD 연동BYOK
💜VS Code확장 프로그램으로 통합BYOK
MCP에서는 왜 무료인가?
MCP 환경에서는 이미 AI 에이전트(Gemini, Claude 등)가 동작하고 있기 때문에, 진단 결과를 보고 에이전트가 직접 코드를 수정합니다. 별도 API 키가 필요 없습니다.

대시보드, CLI, VS Code처럼 AI 에이전트 없이 단독 사용할 때는 자신의 API 키를 등록(BYOK)하여 자동 수리를 사용합니다.
···

BYOK 자동 수리

BYOK(Bring Your Own Key)는 자신의 AI API 키를 등록하여, 대시보드나 CLI에서 실패한 진단을 자동으로 수리하는 기능입니다.

지원 프로바이더
OpenAIAnthropicGoogle GeminiOpenRouter
수리 흐름
1진단 실행 → ERROR/WARNING 발견
2소스코드 + 진단 컨텍스트를 AI에 전달
3AI가 수정사항 생성 → .bak 백업 후 적용
4재진단으로 수리 결과 검증
보안: API 키는 로컬 config.json에만 저장되며, .gitignore에 자동 추가됩니다. 수리 전에는 반드시 .bak 백업이 생성됩니다.
···

진단 파일 예시

.vibe-diagnosis/diagnostics/ 폴더에 .diag.js 파일을 생성합니다.

module.exports = { id: 'task-001-user-login', name: 'User Login Flow', layer: 'TASK', linkedTask: 'TASK-001', async run(ctx) { const auth = require('../src/auth'); const result = auth.login( 'test@test.com', 'password123' ); if (!result.token) { return { status: 'ERROR', details: 'Login did not return token' }; } return { status: 'OK', details: 'Login flow verified' }; } };

MCP 환경에서는 AI 에이전트가 작업을 완료할 때마다 이런 진단 파일을 자동으로 생성합니다.
